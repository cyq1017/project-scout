# Source Profile: Market Opportunity

## Use When

Use this profile when deciding whether a new capability, workflow, or skill
addresses a real market or operator need.

## Required Sources

| Source | Required? | Reason |
| --- | --- | --- |
| GitHub | yes | Finds open-source implementations and developer interest. |
| Web/product pages | yes | Finds commercial positioning and existing categories. |
| Community sources | optional | Useful for pain-language and unmet workflow signals. |
| Job market | optional | Useful when the opportunity depends on team roles or hiring demand. |
| Papers | no | Only required for research-heavy markets. |

## Query Seeds

- problem phrase + "tool"
- workflow phrase + "automation"
- target user + "pain point"
- category term + "open source"
- category term + "alternative"

## Stop Criteria

- At least one open-source path and one product/category path are represented,
  or the blind spot is explicitly recorded.
- Known candidates from the project brief are included.
- Two expanded query rounds produce no new high-overlap category or project.

## Blind Spots To Name

- Demand signals may be anecdotal without live market data.
- Private/internal tools are usually invisible.
- Community sources can overrepresent vocal early adopters.
