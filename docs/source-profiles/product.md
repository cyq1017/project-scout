# Source Profile: Product

## Use When

Use this profile when comparing a product idea, SaaS tool, or commercial
alternative before a build/buy/wrap decision.

## Required Sources

| Source | Required? | Reason |
| --- | --- | --- |
| Web/product pages | yes | Captures positioning, packaging, and visible workflows. |
| GitHub | optional | Required when open-source alternatives may satisfy the need. |
| Package registries | optional | Useful for SDKs, CLIs, and developer-facing products. |
| Community sources | optional | Useful for complaints, migration reasons, and adoption signals. |
| Papers | no | Only required for research-heavy products. |

## Query Seeds

- product category + "alternative"
- product category + "open source"
- workflow phrase + "software"
- competitor name + "github"
- competitor name + "pricing"

## Stop Criteria

- The report includes direct alternatives and at least one adjacent category.
- Pricing/packaging is either summarized or named as a blind spot.
- Open-source substitutes are included or explicitly ruled out.

## Blind Spots To Name

- Product claims can change quickly.
- Pricing and enterprise terms may be unavailable.
- Private customer sentiment is not visible from public sources.
