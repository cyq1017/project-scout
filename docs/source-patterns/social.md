# Social Source Pattern

## Use When

Use social or community sources for pain-language, adoption anecdotes, migration
reasons, operational complaints, and early signals that are not visible in
official docs.

## Routes

| Need | Preferred Route |
| --- | --- |
| Developer discussions | Reddit, Hacker News, V2EX, GitHub Discussions |
| Product feedback | public forums and community posts |
| Regional signals | platform-specific search with explicit source notes |
| Validation | cross-check claims against primary docs or repositories |

## Metadata To Capture

- source platform and URL
- date or recency
- quoted pain phrase or summary
- product/project names mentioned
- whether the source is official, user-generated, or anecdotal

## Pitfalls

- Social evidence is anecdotal unless corroborated.
- Viral posts can overrepresent edge cases.
- Platform search can be personalized or incomplete.
- Do not treat user comments as factual claims without source checks.
