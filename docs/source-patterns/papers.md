# Papers Source Pattern

## Use When

Use paper sources for research methods, benchmarks, datasets, algorithms, and
claims that depend on academic or technical publications.

## Routes

| Need | Preferred Route |
| --- | --- |
| Known paper lookup | arXiv, DOI, publisher page, or Hugging Face paper page |
| Implementation discovery | paper title or method name plus `github` |
| Dataset or benchmark context | paper references, project page, benchmark docs |
| Reproduction signals | implementation issues, replication notes, follow-up papers |

## Metadata To Capture

- title, authors, and publication date
- paper URL and project page when available
- method or benchmark name
- linked implementation repositories
- claimed dataset, metric, or evaluation setting
- license or usage constraints for code/data

## Pitfalls

- Paper claims may not reproduce in the target environment.
- Implementations can lag behind the paper revision.
- Datasets and benchmark terms may block commercial or hosted use.
- Citation count is not a fit signal by itself.
