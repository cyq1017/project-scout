# Job-Market Source Pattern

## Use When

Use job-market sources when a decision depends on role demand, operational
ownership, buyer/user language, or whether a workflow maps to real team
responsibilities.

## Routes

| Need | Preferred Route |
| --- | --- |
| Role demand | public job search with bounded queries |
| User language | job descriptions and responsibility bullets |
| Tool mentions | skill/tool names in job requirements |
| Market adjacency | roles hiring for similar workflows |

## Metadata To Capture

- role title and company type
- query used
- workflow or tool phrases
- required skills or systems
- geography or seniority if relevant
- source URL and access date

## Pitfalls

- Job posts are noisy and can be duplicated.
- Hiring demand does not prove product demand.
- Public job listings underrepresent internal tooling.
- Do not store personal applicant data or scrape private job boards.
